# Image URL Production Fix

## Problem
Image URLs stored in the database from development were hardcoded with `http://127.0.0.1:8000`:
```
http://127.0.0.1:8000/uploads/b038802f512642648bfbf7655ee628a0.jpg
```

In production, these URLs don't change automatically, so images fail to load.

## Solution
Created a utility function that **dynamically rewrites image URLs** based on the current deployment.

### How It Works

**New file:** `app/utils/image_url.py`
- `normalize_image_url()` function extracts the filename from hardcoded URLs
- Reconstructs the URL using current `API_BASE_URL` from environment
- Works with:
  - Hardcoded localhost URLs (development)
  - Production URLs (Railway)
  - Relative paths (fallback)

### Example

**Database has:**
```
http://127.0.0.1:8000/uploads/b038802f512642648bfbf7655ee628a0.jpg
```

**Function extracts:** `b038802f512642648bfbf7655ee628a0.jpg`

**With `API_BASE_URL=https://your-app.railway.app`, returns:**
```
https://your-app.railway.app/uploads/b038802f512642648bfbf7655ee628a0.jpg
```

## How It Works Under the Hood

### Development (localhost)
```
Database: http://127.0.0.1:8000/uploads/abc123.jpg
API_BASE_URL: http://localhost:8000
Returns: http://localhost:8000/uploads/abc123.jpg ✅
```

### Production (Railway)
```
Database: http://127.0.0.1:8000/uploads/abc123.jpg
API_BASE_URL: https://myapp.railway.app
Returns: https://myapp.railway.app/uploads/abc123.jpg ✅
```

### No API_BASE_URL (Fallback)
```
Database: http://127.0.0.1:8000/uploads/abc123.jpg
API_BASE_URL: None
Returns: /uploads/abc123.jpg (relative, uses request host) ✅
```

## API_BASE_URL Requirement

**For image URLs to work correctly in production:**
- Set `API_BASE_URL` in Railway variables to your full backend URL
- Example: `https://your-app-xyz123.railway.app`
- Without it, relative paths are used (less reliable)

## Troubleshooting

### Images Still Show Old URL
**Cause:** API_BASE_URL not set in Railway
**Fix:** Set `API_BASE_URL=https://your-railway-app.up.railway.app` in variables

### Images Show as Broken
**Cause:** Relative path not working in CORS environment
**Fix:** Set full `API_BASE_URL` instead of relying on fallback

### Images Load Locally but Not in Production
**Cause:** API_BASE_URL mismatch
**Fix:** Verify API_BASE_URL matches your actual Railway domain
