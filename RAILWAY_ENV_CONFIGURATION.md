# Railway Environment Variables Configuration

## Problem Resolved
**Error:** `pydantic_core._pydantic_core.ValidationError: 1 validation error for Settings - api_base_url - Field required`

**Root Cause:** `api_base_url` was marked as required but not set in Railway environment.

**Solution:** Made `api_base_url` optional with intelligent defaults.

---

## Updated Settings Configuration

### app/config.py Changes

**BEFORE:**
```python
api_base_url: str = Field(..., env="API_BASE_URL")  # REQUIRED - crashes without it
cors_origins: str = Field(None, env="CORS_ORIGINS")
```

**AFTER:**
```python
api_base_url: Optional[str] = Field(None, env="API_BASE_URL")  # Optional
cors_origins: Optional[str] = Field(None, env="CORS_ORIGINS")  # Optional

@property
def cors_origins_list(self) -> List[str]:
    # Fallback chain: CORS_ORIGINS → API_BASE_URL → localhost default
    if self.cors_origins:
        value = self.cors_origins.strip()
    elif self.api_base_url:
        value = self.api_base_url.strip()
    else:
        value = "http://localhost:5173"  # Safe default
```

## Complete Railway Environment Variables

### Required Variables (MUST BE SET)

| Variable | Example Value | Description |
|----------|---------------|-------------|
| `DATABASE_URL` | `postgresql+psycopg://user:pass@host/dbname` | PostgreSQL connection string from Railway Postgres plugin |
| `SECRET_KEY` | `your-secret-key-min-32-chars-here` | JWT signing key (min 32 characters for production) |

### Optional Variables (Recommended)

| Variable | Default | Description |
|----------|---------|-------------|
| `API_BASE_URL` | `None` (derived at runtime) | Full URL of the API (e.g., `https://your-app.up.railway.app`) |
| `CORS_ORIGINS` | Uses `API_BASE_URL` or frontend default | Comma-separated CORS allowed origins |

### Derived/Automatic Variables

| Variable | Auto-Set | Value |
|----------|----------|-------|
| `ALGORITHM` | ✅ | `HS256` (JWT algorithm) |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | ✅ | `30` (token expiration) |
| `APP_NAME` | ✅ | `Project Bheem API` |

## Railway Configuration Steps

### 1. Connect PostgreSQL Database

In Railway dashboard:
1. Go to **Plugins** → **Add Plugin**
2. Select **PostgreSQL**
3. Railway automatically creates `DATABASE_URL`
4. Copy the connection string (it will be set as env var)

### 2. Set Required Environment Variables

In Railway dashboard → **Variables** section:

```
# REQUIRED
DATABASE_URL=postgresql+psycopg://...  (auto-generated by Railway)
SECRET_KEY=your-secret-key-here

# OPTIONAL but RECOMMENDED
API_BASE_URL=https://your-app.up.railway.app
CORS_ORIGINS=https://your-frontend.vercel.app
```

### 3. Verify Environment Variables are Set

```bash
# Check on Railway dashboard
Settings → Variables → Confirm all variables are visible
```

## Security Best Practices

### ✅ DO:
- Use **strong SECRET_KEY** (min 32 characters)
- Rotate SECRET_KEY regularly
- Use different keys for dev/prod
- Never commit SECRET_KEY to git
- Use Railway's variable encryption

### ❌ DON'T:
- Use default values like "secret-key"
- Hardcode URLs in code (use env vars)
- Commit .env file to git
- Share SECRET_KEY across environments

### Generate Secure SECRET_KEY

```bash
# Python
python -c "import secrets; print(secrets.token_urlsafe(32))"

# Linux/Mac
openssl rand -hex 32
```

## Troubleshooting

### Issue: "ValidationError: api_base_url - Field required"
**Status:** ✅ FIXED (with this update)
- Solution: `api_base_url` is now optional
- Fallback: Uses `CORS_ORIGINS` or safe default

### Issue: CORS Error from Frontend
**Cause:** Frontend URL not in `CORS_ORIGINS`
**Fix:** Add frontend URL to `CORS_ORIGINS` variable

```
CORS_ORIGINS=https://frontend.vercel.app
```

### Issue: Database Connection Failed
**Cause:** Invalid `DATABASE_URL`
**Fix:** Verify Railway PostgreSQL plugin is connected
1. Go to Railway dashboard
2. Check Postgres service is running
3. Copy full `DATABASE_URL` from variables

### Issue: Frontend Can't Connect to Backend
**Cause:** `VITE_API_BASE_URL` points to wrong URL
**Fix:** Ensure frontend has correct backend URL

```
VITE_API_BASE_URL=https://your-railway-backend.up.railway.app
```
